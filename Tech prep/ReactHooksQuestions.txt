1. What are React Hooks :
      -> Hooks are functions that allows to  use state and lifecycle features in functional components (no need for class components).
      -  Before Hooks → only class components could use state
      -  After Hooks → functional components can do everything 

        Why Hooks?
          - Cleaner code
          - Easier to reuse logic
          - No 'this' confusion
          - Better for modern React apps
2. Hooks Definition
  Core React Hooks :
       useState – Manages state in a functional component.
       useEffect – Handles side effects like API calls, subscriptions, and DOM updates.
       useContext – Accesses global state without prop drilling.
 Additional Built-in Hooks
       useRef – Stores mutable values without causing re-renders and accesses DOM elements.
       useMemo – Memoizes computed values to optimize performance.
       useCallback – Memoizes functions to prevent unnecessary re-creations.
       useReducer – Manages complex state logic using a reducer function.
       useLayoutEffect – Runs synchronously after DOM updates but before the browser paints.
       useImperativeHandle – Customizes the instance value exposed to parent components via refs.
       useDebugValue – Displays custom labels for hooks in React DevTools.
React 18+ Hooks
       useId – Generates unique IDs for accessibility and SSR compatibility.
       useTransition – Marks updates as non-urgent to keep UI responsive.
       useDeferredValue – Defers updating a value to improve performance.
       useSyncExternalStore – Subscribes to external stores safely (used in state libraries).
       useInsertionEffect – Runs before DOM mutations (mainly for CSS-in-JS libraries).
       Custom Hook – A reusable function that encapsulates and shares hook logic across components.

What problem do Hooks solve?

👉 They eliminate:

 - Class components complexity
 - this keyword confusion
 - Code duplication (logic reuse)
3. Explain useState with example.

👉 Used to manage state in functional components

const [count, setCount] = useState(0);
4. How does useEffect work?

👉 Handles side effects (API calls, subscriptions)

useEffect(() => {
  fetchData();
}, []);
5. What are dependencies in useEffect?

👉 Controls when effect runs

[] → run once
[state] → run on state change
no array → run always
6. What is cleanup function in useEffect?

 - A cleanup function in useEffect is a function that runs before the component is removed (unmounted)
or before the effect runs again, to clean up side effects.
   (or)
- Cleanup function in useEffect is used to remove side effects like event listeners, timers, or subscriptions to avoid memory leaks.”

useEffect(() => {
  return () => console.log("cleanup");
}, []);


⚡ Advanced Hooks Questions
7. Difference between useEffect and useLayoutEffect?
“I use useEffect for most side effects, but useLayoutEffect when I need to read or modify the DOM
before the browser paints to avoid flickering.”


Feature	   |  useEffect	   |useLayoutEffect
Execution time |	 After paint   |	Before paint
Nature         |	Asynchronous   |	Synchronous
UI blocking	   | ❌ No           |	✅ Yes (blocks rendering)
Performance	   | Better	          | Can impact performance
Use case       | 	API calls, subscriptions  | DOM measurements, layout changes


🧠 How to Think About It

👉 useEffect

Runs later
Doesn’t block UI
User may briefly see un-updated UI

👉 useLayoutEffect

Runs immediately
Blocks UI until done
Prevents visual flicker


8. What is useRef used for?

useState	useRef

10. What is useMemo?

👉 Optimizes expensive calculations

const value = useMemo(() => compute(data), [data]);
11. What is useCallback?

👉 Memoizes functions

const fn = useCallback(() => {}, []);
12. Difference between useMemo and useCallback?
useMemo	useCallback
Returns value	Returns function
13. When should you NOT use useMemo?

👉 When computation is cheap (avoid over-optimization)

⚠️ Rules & Best Practices
18. What are Rules of Hooks?
Only call at top level
Only inside React functions
Not inside loops/conditions
19. What are common mistakes with Hooks?

Overusing useMemo
Infinite loops
Updating state incorrectly

21. Difference between Redux and contextApi :
Context API is a simple built-in state sharing feature, while Redux is a powerful external library 
for managing complex global state with strict patterns.

Comparison Table
Feature    |	Context API 	     |Redux
Type	     | Built-in React feature      |	External library
Complexity |	Simple	           |     Complex
Setup      | 	Minimal                |  	More boilerplate
State management	 | Basic global state  | 	Advanced state management
Performance  |	Can cause re-renders   | Optimized (fine-grained updates)
Debugging	 | Limited	                 | Powerful (Redux DevTools)
Middleware support |	❌ No            |	✅ Yes (Thunk, Saga)
Best for	       |Small to medium apps	| Large-scale apps

22. Controlled components vs unControlled components :
  -  Controlled components are managed by React state, while uncontrolled components are managed by the DOM itself.

Feature	       | Controlled Component  |	Uncontrolled Component
Source of truth    |	React state	     | DOM
Value handling     |	value + onChange | ref
Re-render on change |	✅ Yes	     | ❌ No
Control	        | Full control by React |	Less control
Validation	        | Easy	              | Harder
Use case	        | Forms with validation  |	Simple forms / quick inputs

23. React Fragments :
   - React Fragments allow grouping multiple elements without adding extra DOM nodes.
 Wrong Example : // unwantedly added div to wrap
    return (
  <div>
    <h1>Hello</h1>
    <p>World</p>
  </div>
);

using Fragment : // shorter version
 return (
  <>
    <h1>Hello</h1>
    <p>World</p>
  </>
);

Full syntax :
import React from "react";

<React.Fragment>
  <Child1 />
  <Child2 />
</React.Fragment>
