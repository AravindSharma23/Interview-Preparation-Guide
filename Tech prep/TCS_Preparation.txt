1.Difference between inline and block elements in html :
BLOCK :
 - Always start on a new line
 - Take up the full width available (by default)
 - Can contain inline elements and other block elements
INLINE : 
 - Do not start on a new line
 - Only take up as much width as needed
 - Typically contain text or other inline elements
 - Cannot contain block elements

11. VAR, LET,CONST

 var : hoisted + initialized = undefined
  -  Function scoped
  - Can be re-declared and updated
  - Gets hoisted and initialized with undefined 
Let :  hoisted + initialized (available in TDZ)= throws reference err
  - Block scoped ({ })
  - Can be updated, but NOT re-declared in same scope
  - Hoisted but in Temporal Dead Zone (TDZ)
Const : hoisted + initialized (available in TDZ)= throws reference err
 - Block scoped
 - Cannot be updated or re-declared
 - Must be initialized at declaration
 - Hoisted but in Temporal Dead Zone (TDZ)

    
12 . Temporal Dead Zone :
  The TDZ is the time between:
          -  When a variable is hoisted And when it is initialized
    - During this time, accessing the variable throws an error

13. Hoisting :
  Hoisting is JavaScript’s default behavior where declarations are moved to the top of their scope.
  -  var : hoisted + initialized = undefined
  - let & Const : hoisted + initialized (available in TDZ) = throws reference err

15. call() :
     Calls function immediately.
     pass args one by one 
  Eg :
    function greet(city) {
        console.log(this.name + " from " + city);
    }

    const user = { name: "Aravind" };

    greet.call(user, "Salem");

   apply() :
    - same as call
     - args passed as array
    Eg:  greet.apply(user, ["Chennai"]);

  bind() :
   - Doesn't call immediately
   - Returns a new function
  Eg : 
   const fn = greet.bind(user, "Bangalore");
    fn();

17. promise :
     promise is an object that is a result of async operation (either success or failure).
   Promise States :
   - pending - initial state
   - fulfilled - success(resolve)
   - rejected - failure(reject)
  Promise Methods :
   -> 1. Promise.all():
        - Wait for all promises to successed
        - If one fails → entire thing fails
    Eg :
const p1 = Promise.resolve("User Data");
const p2 = Promise.resolve("Posts Data");
const p3 = Promise.resolve("Comments Data");

Promise.all([p1, p2, p3])
  .then((result) => {
    console.log(result);
  })
  .catch((error) => {
    console.log(error);
  });
O/P : ["User Data", "Posts Data", "Comments Data"]

  ->2. Promise.race() :
     - Returns the FIRST completed promise
     - It may be (success OR failure)
   Eg:
  const p1 = new Promise((resolve) =>
  setTimeout(() => resolve("Fast API"), 1000)
);

const p2 = new Promise((resolve) =>
  setTimeout(() => resolve("Slow API"), 3000)
);

Promise.race([p1, p2])
  .then((result) => console.log(result));

O/P : Fast API

->3. Promise.allSetteled() :
   - Waits for ALL promises
   - Doesn’t fail even if some reject.
Eg : 
const p1 = Promise.resolve("User API");
const p2 = Promise.reject("Post API Failed");
const p3 = Promise.resolve("Comment API");

Promise.allSettled([p1, p2, p3])
  .then((result) => console.log(result));
O/P :
 [
  { status: "fulfilled", value: "User API" },

  { status: "rejected", reason: "Post API Failed" },

  { status: "fulfilled", value: "Comment API" }
]
->4. Promise.any() :
    - Returns FIRST SUCCESSFUL promise
    - Ignores rejected ones.
Eg :
const p1 = Promise.reject("Server 1 Failed");

const p2 = new Promise((resolve) =>
  setTimeout(() => resolve("Server 2 Success"), 2000)
);

const p3 = Promise.reject("Server 3 Failed");

Promise.any([p1, p2, p3])
  .then((result) => console.log(result))
  .catch((error) => console.log(error));

O/P : Server 2 Success

18. Debouncing &  Throttling:
     A function that delays execution until user stops triggered event.
   Eg : User typing searchbox
        without Debouncing - API called every key stroke 
        with Debouncing - API called only once
   function debounce(fn,delay){
        let timer;
       return function(...args){
        clearTimeout(timer);
          timer = setTimeout(()=>{
            fn.apply(this,args);
          },delay)
         }
     }
  const handleSearch = debounce((value)=>{
     console.log(value)
  },1000)

Throttling :
    A function called at most once in a fixed time interval
   Eg : 
      User scrolls continously 
      You want function every 1 second only
