1. What is Next.js?

Answer:
Next.js is a React framework that provides features like:

Server-Side Rendering (SSR)
Static Site Generation (SSG)
File-based routing
API routes
Built-in performance optimizations

It helps build SEO-friendly and fast web applications.

2. What is the difference between CSR, SSR, and SSG?

Answer:

CSR (Client-Side Rendering): Rendering happens in browser (React default)
SSR (Server-Side Rendering): Page generated on each request (via getServerSideProps)
SSG (Static Site Generation): Pages generated at build time (getStaticProps)

👉 SSR = dynamic data
👉 SSG = fast + cacheable

3. What is file-based routing in Next.js?

Answer:
In Next.js, routing is based on the file structure inside the pages or app folder.

Example:

pages/about.js → /about
pages/blog/[id].js → /blog/1
4. What are dynamic routes?

Answer:
Dynamic routes allow you to create pages with parameters.

Example:

pages/product/[id].js

Access:

const router = useRouter();
console.log(router.query.id);
5. What is getServerSideProps?

Answer:
Used for SSR. It fetches data on every request.

export async function getServerSideProps() {
  const data = await fetch("API");
  return { props: { data } };
}
6. What is getStaticProps?

Answer:
Used for SSG. Data is fetched at build time.

export async function getStaticProps() {
  return { props: { data } };
}
7. What is getStaticPaths?

Answer:
Used with dynamic routes in SSG to define which pages to pre-render.

8. What is Incremental Static Regeneration (ISR)?

Answer:
Allows updating static pages after deployment.

return {
  props: { data },
  revalidate: 10
}

👉 Page updates every 10 seconds.

9. What are API routes in Next.js?

Answer:
Next.js allows backend logic inside pages/api.

Example:

pages/api/user.js
export default function handler(req, res) {
  res.status(200).json({ name: "John" });
}

26 . ISR - Incremental Static Regeneration
     “ISR allows you to update static pages after deployment without rebuilding the entire site.”
      🧠 Simple Idea
SSG → builds once 🏗️
ISR → builds + updates later 🔁

👉 Best of SSG + SSR combined
Eg : 
export async function getStaticProps() {
  const res = await fetch("https://api.example.com/data");
  const data = await res.json();

  return {
    props: { data },
    revalidate: 10, // every 10 seconds it updates 
  };
}
27 . App router vs Page router :
    App Router is the newer routing system in Next.js that supports server components, layouts, and modern data fetching, while Pages Router is the older approach using getStaticProps and getServerSideProps. App Router is now recommended for new applications.
   | App Router                                  | Pages Router             |
| ------------------------------------------- | ------------------------ |
| `page.tsx` is required                      | Any file becomes a route |
| Special files (`layout.tsx`, `loading.tsx`) | No special structure     |

28 . Explain Next.js caching & rendering strategy :
   - “Next.js optimizes performance using multiple caching layers (data, full route, router cache) and flexible rendering strategies (SSG, SSR, ISR, CSR), controlled mainly through the fetch API and route configuration.”
    - Explain  about Rendering strategies SSR,SSG,CSR,ISR .
  Caching Layers :
     1. 🧠 Data Cache

👉 Caches API responses (fetch)

Example
await fetch("https://api.com/data"); // cached
Control it:
cache: "force-cache" (default)
cache: "no-store"
next: { revalidate: 10 }
2. 📄 Full Route Cache

👉 Stores the entire rendered HTML

Used for static pages
Served instantly from cache
3. ⚡ Router Cache (Client-side)

👉 Stores previously visited routes in browser

Faster navigation
No refetch needed
