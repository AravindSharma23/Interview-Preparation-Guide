1. Why TypeScript instead of JavaScript?
    “TypeScript is a superset of JavaScript that adds static typing, which helps catch errors at compile time, improves code readability,
and makes large-scale applications easier to maintain.”

  Key Points to Add:
 - Early error detection
 - Better IDE support (autocomplete, refactoring)
 - Improved code scalability
 - Safer code with types

2. Interface vs type :
  -  “Both are used to define types in TypeScript, but interfaces are mainly for object shapes and support extension,
while types are more flexible and can represent unions, primitives, and complex structures.”

| Feature     | Interface        | Type                              |
| ----------- | ---------------- | --------------------------------- |
| Extending   | `extends`        | `&` (intersection)                |
| Flexibility | Less             | More                              |
| Use case    | Objects, classes | Unions, primitives, complex types |

3. Generics :
   - “Generics allow us to create reusable components or functions that work with multiple data types while maintaining type safety.”

  function identity<T>(value: T): T {
  return value;
}

4. Optional Chaining :
  - “Optional chaining (?.) allows safe access to nested object properties without throwing errors if a value is null or undefined.”
 Eg :  user?.address?.city

5.  any vs unknown :
   - “any disables type checking completely, while unknown is a safer alternative that requires type checking before usage.”
| Feature        | any    | unknown          |
| -------------- | ------ | ---------------- |
| Type safety    | ❌ None | ✅ Safe           |
| Usage          | Direct | Needs type check |
| Recommendation | Avoid  | Prefer           |

Eg : 
 let value: unknown = "hello";

if (typeof value === "string") {
  console.log(value.toUpperCase()); // safe
}

6 . What is Type Inference?

👉 TypeScript automatically determines types without explicit annotation.

7 . What is Union Type?
  - Variable can hold multiple types.
  Eg :     let value: string | number;

8 . What is Intersection Type?
   -  Combines multiple types into one.
   Eg  :
      type A = { name: string };
type B = { age: number };

type C = A & B;

9. What is Type Assertion? 
  -  Tells TypeScript about the type explicitly.
  Eg :  let value = "hello" as string;

10. enum :
  - Enum is a way to define a set of named constants. It makes your code more readable .
 enum Direction {

}

✅ User roles

11 . What are Utility Types in TypeScript?
    - Utility types are built-in generic types that help transform and manipulate existing types easily.
     1. Partial<T>

👉 Makes all properties optional

type User = {
  name: string;
  age: number;
};

type PartialUser = Partial<User>;

✅ Result:

{
  name?: string;
  age?: number;
}
2. Required<T>

👉 Makes all properties required

type User = {
  name?: string;
};

type RequiredUser = Required<User>;
3. Readonly<T>

👉 Makes properties immutable

type User = {
  name: string;
};

const user: Readonly<User> = { name: "John" };
// user.name = "Doe"; ❌ Error
4. Pick<T, K>

👉 Select specific properties

type User = {
  name: string;
  age: number;
};

type NameOnly = Pick<User, "name">;
5. Omit<T, K>

👉 Remove specific properties

type User = {
  name: string;
  age: number;
};

type WithoutAge = Omit<User, "age">;
6. Record<K, T>

👉 Create object type with keys and values

type Roles = "admin" | "user";

const rolePermissions: Record<Roles, boolean> = {
  admin: true,
  user: false,
};
7. Exclude<T, U>

👉 Removes types from union

type A = string | number;
type B = Exclude<A, number>; // string
8. Extract<T, U>

👉 Extracts common types

type A = string | number;
type B = Extract<A, string>; // string
9. NonNullable<T>

👉 Removes null and undefined

type A = string | null | undefined;
type B = NonNullable<A>; // string
